# claude-monitor ‚Äî next session pickup list

Ordered by value. Each item is self-contained enough to tackle in isolation.

## High value

- [x] **Tauri window wrapper** ‚Äî `crates/cm-app/src-tauri/` is a workspace member building the `cm-app-tauri` binary. The daemon from `cm-app` is now a library (`cm_app::start()`); the Tauri binary boots the same daemon on a background Tokio runtime, opens a webview at `http://127.0.0.1:<bound-port>/`, and registers a tray icon with Show/Hide/Quit. Closing the window hides it so the daemon keeps running in the tray. Placeholder indigo icons generated by `icons/gen.py` (Python stdlib only; regenerate if designing a real icon). **Follow-up:** the real icon, bundling (`cargo tauri build` needs tauri-cli), and autostart (separate TODO) still need doing.
- [x] **Auto-start on login** ‚Äî `cm install` (macOS only) writes `~/Library/LaunchAgents/com.claude-monitor.daemon.plist` pointing at the sibling `cm-app` binary (override with `--binary`), runs `launchctl load -w`, and merges a `statusLine` entry into `~/.claude/settings.json` (`{type:"command", command:"<cm> statusline"}`). `cm uninstall` unloads + removes the plist and drops the `statusLine` only if it still points at our `cm` binary. Logs go to `~/.claude/claude-monitor/cm-app.{log,err}`. **Follow-up:** an equivalent story for `cm-app-tauri` (tray version) ‚Äî right now install targets the headless daemon only.
- [x] **Cost per-turn estimate** ‚Äî `cm-core::pricing` holds a prefix-matched price table (Opus/Sonnet/Haiku 3.5/4.x at published per-token rates). `turns.estimated_cost_usd` is computed on insert and backfilled on first open of an existing DB. `GET /v1/sessions` returns `estimated_cost_usd` (session sum) alongside the snapshot's `last_cost_usd`; `GET /v1/sessions/:id/turns` returns it per row. Sessions table in the UI shows both columns side-by-side. **Follow-up:** model IDs not matching `opus|sonnet|haiku` leave cost `NULL` ‚Äî add a fallback or explicit list when new model families ship.
- [x] **Session detail view** ‚Äî hash router in `app.js` handles `#/live`, `#/sessions`, `#/trends`, `#/session/:id`. Session rows are clickable and navigate to the detail view, which renders the full per-turn stacked chart plus a snapshot timeline (cost on the left axis, context % and 5-hour % on the right axis in dashed form). Live WS `turn` events for the detail session update the chart in place. **Follow-up:** the snapshot chart uses raw ts; if statusline never ran for a session it shows an empty-state hint rather than silently rendering nothing.

## Medium value

- [x] **Trend aggregates server-side** ‚Äî `GET /v1/trends?window={hour|day|week}` groups `turns.ts` in SQL, returning `{ts, total_tokens, total_input_tokens, total_output_tokens, total_cache_read, total_cache_creation, total_cost_usd, turns}` per bucket. UI `loadTrends()` now calls the endpoint directly instead of summing session rows in the browser. **Follow-up:** window=month would need day-of-month arithmetic (SQLite `strftime`) rather than fixed-size buckets; not worth it yet.
- [ ] **Rate-limit alerting** ‚Äî push-notify when `five_hour_pct` crosses 80/95. Out of scope for the daemon; emit a desktop notification via `notify-rust` when wrapped in Tauri.
- [x] **Subagent row breakdown** ‚Äî `cm subagent-statusline` reads Claude Code's `subagentStatusLine` stdin payload (`{columns, tasks[]}`), emits one `{"id":‚Ä¶,"content":‚Ä¶}` JSON line per task back to Claude Code, and forwards the raw JSON to `POST /v1/subagent-event` (session id resolved from body or `?session_id=` query fallback). Server upserts into a new `subagent_tasks` table keyed on `(session_id, task_id)` ‚Äî merges fields via `COALESCE(excluded, existing)` so a later event without a field doesn't wipe the previous value, preserves `first_seen_at`, advances `last_seen_at`. `GET /v1/sessions/:id/subagents` lists rows; WS broadcasts `{kind:"subagent_snapshot", session_id, ts_ms, tasks}` for live updates. Session detail view has a "Subagents" panel (task, name, status, tokens, elapsed, cwd, last seen). **Follow-up:** `cm install` does NOT wire the `subagentStatusLine` hook into `~/.claude/settings.json` ‚Äî users have to add it manually (`{"subagentStatusLine":{"type":"command","command":"<cm> subagent-statusline"}}`). Also: no time-series history, only the latest state per task; add a `subagent_task_events` table if retroactive audit becomes useful.
- [ ] **Compaction events** ‚Äî transcript JSONL has `type:"summary"` and compact boundary markers. Mark these on the per-turn chart so the context-window drops make sense.
- [ ] **Export / CSV** ‚Äî `GET /v1/sessions/:id/turns.csv` for spreadsheet analysis.

## Low value / cleanup

- [x] **Tests for cm-app** ‚Äî `crates/cm-app/tests/integration.rs` covers the router via `tower::ServiceExt::oneshot` (health, `/v1/event` happy+bad JSON, subagent with/without session_id, `/v1/trends` bad window, `/v1/sessions` JSON shape), `tailer::ingest_new_bytes` (happy path, partial line, post-truncation rotation), and `drain_queue` against a path-injected temp queue file. Dev-deps: `tempfile`, `http-body-util`.
- [x] **Tests for cm-cli** ‚Äî `crates/cm-cli/tests/cli.rs` uses `assert_cmd` + `predicates` with a sandboxed `HOME` per test. Covers the statusline contract (never blanks, prints `[model]`, `%`, `$cost`, 5h/7d row), subagent rows (one JSON-per-task, label/tokens formatting), `port` (`CM_PORT` env override + "none" fallback), and the offline queue round-trip.
- [ ] **Tighten CORS** ‚Äî `tower_http::cors::CorsLayer::permissive()` is fine for localhost-only, but tighten to `allow_origin(Any)` stripped of credentials when the Tauri wrapper lands.
- [x] **Config file** ‚Äî `~/.claude/claude-monitor/config.toml` parsed by `cm_core::config::load`. Keys: `port`, `retention_days`, `include_globs`, `exclude_globs` (all optional, `deny_unknown_fields` so typos fail fast). `PathFilter::from_config` compiles the globs; the tailer consults it on both initial drain and live events. `CM_PORT` env wins over `config.port`, which wins over the auto-bind default. **Follow-up:** globs match against the absolute path of the transcript file; the easiest patterns target the Claude-encoded project-dir segment (e.g., `*/projects/-Users--vpon-foo/*`). No test coverage yet for glob matching itself.
- [x] **Retention** ‚Äî `config.retention_days` (>0) starts a background task that calls `cm_core::db::delete_older_than` at startup and every 24h. Removes `turns`/`snapshots`/`subagent_tasks` rows past the cutoff, plus `sessions` that become orphaned. Logs a one-liner whenever anything is deleted. **Follow-up:** no `VACUUM` ‚Äî SQLite will reuse the pages, but the file won't shrink. If reclaiming disk matters, add an opt-in `vacuum_on_sweep`.
- [ ] **Windows support** ‚Äî `paths.rs` hard-codes POSIX conventions; `~/.claude/projects/<encoded-path>` decoding uses `/` which is wrong on Windows. Not urgent unless you actually use Windows.
- [x] **Upgrade-safe schema migrations** ‚Äî `cm_core::db::migrate` now tracks applied versions via `PRAGMA user_version`. The base schema still runs idempotently on every open (unchanged); incremental migrations live in a `MIGRATIONS: &[(&str, fn(&Connection))]` list ordered oldest‚Üínewest, each applied exactly once and then stamped. Migration v1 is the legacy `ALTER TABLE turns ADD COLUMN estimated_cost_usd` + backfill, so pre-framework databases upgrade transparently on next launch. **Follow-up:** append-only ‚Äî never reorder or delete an entry, or already-migrated DBs will silently skip steps.

## Known quirks

- The `decode_project_dir` function in `crates/cm-app/src/tailer.rs` is a heuristic: Claude Code encodes `/` ‚Üí `-` and `-` ‚Üí `--`. Edge cases (e.g. paths with embedded `--`) may decode wrong, but it's only cosmetic (used for display, not joining).
- The app watches `~/.claude/projects` recursively. If you have a very large backfill (thousands of sessions), the initial ingest on first launch can take a few seconds. Subsequent launches resume from `tail_state.offset` per file, so it's fast.
- `Cargo.lock` is gitignored (see `.gitignore`); this is a workspace of binaries, but you may want to commit it for reproducible builds ‚Äî toggle as you prefer.

## Open decisions

- **Single daemon vs per-user**: today it's per-user, listening on localhost. If you ever want a shared dashboard across a team, swap SQLite for Postgres and bind to a non-loopback interface behind auth.
- **Historical import depth**: the tailer ingests every pre-existing JSONL on first run. That's convenient but could be surprising for very old transcripts. Consider a `--since` flag.
